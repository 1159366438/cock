## 2. 功能详情

### 2.1.0 v1.0.0 - 基础功能（内测版）
**目标**：搭建核心业务流程，验证技术选型。

**核心功能**：

- [x] 基于模拟数据的用户登录/登出
- [x] 上班打卡功能（记录打卡时间）
- [x] 考勤记录的分页查询
- [x] 基础UI/UX设计（Element Plus）
- [x] 前后端联调与基础异常处理

**技术实现**：
- 后端：Spring Boot + MyBatis实现基础CRUD操作
- 前端：Vue 3 + Element Plus实现基础界面，Axios进行API通信

### 2.2.1 v1.0.1 - 基础功能（修复版）
**目标**：解决登录认证问题，完善安全性，优化代码质量。

**核心功能**：
- [x] 修复BCrypt密码哈希验证问题
- [x] 修复Element Plus废弃API警告
- [x] 优化用户体验
- [x] 解决Git分支管理问题
- [x] 将硬编码值替换为常量（分页参数、错误码、成功消息等）
- [x] 统一错误码管理，创建ErrorCodeConstants类
- [x] 实现ResponseResult的常量化改造（error和success方法）
- [x] 提升代码可维护性和一致性
- [x] 完善日志记录规范
- [x] 修复API文档与代码不一致问题

**技术实现**：
- 后端：BCrypt密码加密算法，统一异常处理和常量管理
- 前端：Element Plus组件升级，常量和错误码统一管理


### 2.2.0 v1.1.0 - 用户与权限管理（内测版）
**目标**：引入组织架构与权限控制系统，实现多用户管理模式。

**核心功能**：
- [x] 用户注册功能（前端表单验证、后端数据存储）
- [ ] 部门管理（增删改查、层级结构支持）
- [ ] 基于角色的访问控制（RBAC权限模型）
- [ ] 多表数据库设计（用户-部门-角色关联）
- [ ] 管理员面板（用户权限分配、角色管理）
- [ ] 登录认证集成真实用户数据

**技术实现**：
- 后端：Spring Security 集成 RBAC 权限控制，MyBatis 实现多表关联查询
- 前端：Vue 3 + TypeScript 实现管理界面，权限控制路由守卫

### 2.3.0 v1.2.0 - 业务规则（内测版）
**目标**：增加考勤规则约束，提升可信度。

**核心功能**：
- [ ] 可配置的打卡规则（上下班时间、迟到/早退阈值）
- [ ] 集成地图API（高德/百度）进行位置验证
- [ ] 规则引擎设计（策略模式）
- [ ] 打卡位置实时验证与记录
- [ ] 合规性检查（如距离公司范围判定）

**技术实现**：
- 后端：规则引擎（策略模式），地图API集成（高德/百度）
- 前端：地理位置获取与验证，规则配置界面

### 2.4.0 v1.3.0 - 数据管理（内测版）
**目标**：满足管理者数据分析和导出需求。

**核心功能**：

- [ ] 高级查询（多条件组合：日期范围、部门、用户、状态）
- [ ] 考勤统计报表（出勤率、迟到次数等）
- [ ] Excel导出功能（支持自定义列）
- [ ] 数据过滤与排序（前端表格交互优化）
- [ ] 可定制的报告格式（预览与打印）

**技术实现**：
- 后端：POI实现Excel导出，复杂查询优化
- 前端：高级表格组件，报表可视化图表

### 2.5.0 v1.4.0 - 性能与部署（内测版）
**目标**：优化性能，简化部署流程。

**核心功能**：

- [ ] Redis集成（缓存热点数据、会话管理）
- [ ] Docker容器化（一键部署脚本）
- [ ] 性能优化（数据库索引、SQL优化、前端懒加载）
- [ ] 负载均衡初步设计（Nginx配置）
- [ ] 生产环境就绪的部署文档

## 4. 质量保证

我们将在每个阶段执行以下质量保证活动：

- **单元测试**：核心业务逻辑覆盖率 ≥ 80%
- **集成测试**：关键业务流程（登录、打卡、查询）全链路验证
- **用户验收测试**：每个内测版本邀请 3-5 名真实用户试用并收集反馈
- **性能测试**：v1.4.0 后执行，模拟 1000 并发用户，响应时间 < 2s
- **安全测试**：v2.0.0 前执行，包括 SQL 注入、XSS、CSRF 防护检查
- **代码审查**：每个 PR 至少 1 人审查，确保代码规范与质量

## 5. 风险评估

| 风险类别 | 风险描述 | 可能性 | 影响 | 应对措施 |
|----------|----------|--------|------|----------|
| **技术** | 地图API集成复杂（如坐标系转换、定位精度） | 中 | 高 | 提前调研API文档，准备备用方案（如高德+百度双支持） |
| **资源** | 核心开发人员业余时间不稳定 | 高 | 中 | 每周制定小目标，允许适当缓冲，并行处理非关键任务 |
| **技术** | 规则引擎设计超出预期难度 | 中 | 中 | 先实现简单配置，后续迭代优化；预留技术攻关时间 |
| **需求** | 用户反馈导致需求变更 | 中 | 低 | 严格控制范围，按优先级实施，非核心需求放入后续版本 |
| **协作** | 团队沟通不畅导致效率下降 | 低 | 中 | 建立周会机制，使用项目管理工具（如GitHub Projects）跟踪进度 |
| **部署** | Docker化过程中环境差异导致问题 | 低 | 低 | 使用多环境测试（开发、测试、生产）提前验证脚本 |
